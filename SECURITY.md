# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch state.

## Reporting a Vulnerability

1. Do not open public issues for suspected vulnerabilities.
2. Report findings privately to the repository owner.
3. Include impact, reproduction steps, and affected paths.

## Secure Development Baseline

1. Input validation is enforced at API boundaries.
2. Dependencies are pinned in `requirements.txt`.
3. CI includes automated test and dependency-scan jobs.
